# PromptPanel 使用示例

更新时间：`2026-05-07`

本文给用户和维护者提供可直接复用的场景示例。更多安装和配置见 [README](../README.md)、[配置说明](./配置说明.md) 和 [运维与排错指南](./运维与排错指南.md)。

## 1. AI 对话 Prompt 复用

适合：ChatGPT、Claude、Gemini、Perplexity。

示例词条：

```text
标题：工程级代码审查
类型：Prompt
标签：review, code, risk
正文：
请以资深工程负责人视角审查下面的代码变更。
优先指出真实 bug、回归风险、缺失测试和上线风险。
输出顺序：严重问题、一般问题、测试建议、总体结论。
不要泛泛而谈。
```

使用路径：

```text
打开 ChatGPT/Claude 输入框 -> 按 ⌥2 -> 搜 review -> 回车
```

如果没有辅助功能权限，PromptPanel 会先复制内容，并提示手动 `Command-V`。

## 2. AI Coding 上下文块

适合：Cursor、Copilot Chat、Aider、VS Code Chat。

示例词条：

```text
标题：本项目边界
类型：说明
标签：project, boundary
正文：
当前项目是本地优先 macOS 桌面应用。
不要引入云同步、团队协作或远端工作流编排。
任何 UI 改动必须先对齐 frontend-draft/。
任何数据库改动必须同步 migration、测试、配置文档和恢复 runbook。
```

建议把不同仓库放在不同项目里：

- `Universal`：通用评审、排障、提交模板。
- `PromptPanel`：本项目边界、架构摘要、发布检查。
- `Client-A`：客户 A 的风格指南和 API 约束。
- `Client-B`：客户 B 的交付格式和术语。

## 4. 终端命令片段

示例词条：

```text
标题：Swift release readiness
类型：代码
标签：swift, release
正文：
swift build
swift test
./scripts/check-docs.sh
./scripts/release-readiness.sh --output-dir /tmp/promptpanel-ready-check
```

使用提示：

- 对命令类词条，建议使用“仅复制”路径：唤出面板后按 `Command-C`，再手动检查后粘贴执行。
- 对破坏性命令，不要做成一回车就可执行的词条。

## 5. 客户或项目隔离

目标：避免不同客户、仓库、产品线之间的 Prompt 串用。

建议：

- 把“可跨项目复用”的词条放入 `通用项目`。
- 把带客户名称、内部架构、私有路径、语气要求的词条放入专用项目。
- 在标题前加稳定前缀，例如 `CR7 / release check`、`PromptPanel / docs gate`。
- 对敏感词条加标签，例如 `#private`、`#client-a`、`#security`。

查询示例：

```text
release #promptpanel
review #security
docs #handoff
```

## 6. 粘贴失败排查示例

现象：回车后没有自动粘贴。

处理顺序：

1. 确认内容是否已经在剪贴板里。
2. 打开主窗口的“运行健康”或最近执行记录。
3. 看 `failure_reason`：
   - `accessibility_not_granted`：授予辅助功能权限。
   - `target_app_not_restored`：目标 app 没有恢复焦点，尝试手动点击输入框后重试。
   - `paste_event_creation_failed`：系统事件创建失败，保留剪贴板兜底。
   - `clipboard_write_failed`：剪贴板写入失败，需要优先排查系统剪贴板。
4. 若只在特定 app 中失败，把目标 app、版本、执行日志记录到 [兼容性回归记录](./兼容性回归记录.md)。

## 7. QA 隔离数据目录

开发者可以用隔离目录启动，避免污染自己的真实 Prompt 库：

```bash
QA_ROOT="$(mktemp -d /tmp/promptpanel-qa.XXXXXX)"
PROMPTPANEL_ALLOW_EXISTING_INSTANCE=1 \
PROMPTPANEL_APP_SUPPORT_DIR="$QA_ROOT/AppSupport" \
PROMPTPANEL_LOGS_DIR="$QA_ROOT/Logs" \
.build/debug/PromptPanel
```

验证点：

- 数据库应创建在 `$QA_ROOT/AppSupport/promptpanel.db`。
- 日志应创建在 `$QA_ROOT/Logs`。
- 退出后可删除整个 `$QA_ROOT`。

## 8. 备份恢复演练

先找到备份：

```bash
ls -lt ~/Library/Application\ Support/PromptPanel/Backups
```

先 dry run：

```bash
./scripts/restore-backup.sh --dry-run ~/Library/Application\ Support/PromptPanel/Backups/<backup>.sqlite
```

恢复到隔离目录验证：

```bash
RESTORE_ROOT="$(mktemp -d /tmp/promptpanel-restore.XXXXXX)"
./scripts/restore-backup.sh --target-dir "$RESTORE_ROOT" ~/Library/Application\ Support/PromptPanel/Backups/<backup>.sqlite
sqlite3 "$RESTORE_ROOT/promptpanel.db" 'PRAGMA integrity_check;'
```

不要直接覆盖真实数据，除非已经关闭所有 PromptPanel 实例并确认备份有效。
