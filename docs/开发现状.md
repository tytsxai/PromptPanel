## 2026-04-20 前端设计稿对齐（本轮）

本轮按「`frontend-draft/` 为唯一基准」完成视觉与交互重做，并补齐了相关数据模型。

- 以 `frontend-draft/`（由本地设计导出目录整理入仓）为前端唯一基准，已落入仓库
- 主界面：移除原 SwiftUI `TabView` 底栏标签，改为顶部自定义标题栏 + 居中「内容库 / 设置」分段控件
- 内容库：改为三栏布局（项目侧栏 200px / 词条列表 360px / 预览面板 flex）
  - 侧栏：项目按钮支持「当前」胶囊、计数、底部维护操作组
  - 中栏：搜索框、过滤芯片（全部 / 类型 / 高频标签）、排序菜单、新建
  - 预览：类型胶囊、置顶胶囊、标题、复制/编辑/置顶/删除主动作、底部元信息（项目 / 标签 / 更新时间）
- 设置：三个子标签（通用 / 备份与数据 / 关于），各组 section 按设计稿样式重排
- 快捷面板：Spotlight 紧凑行布局
  - 行内：序号、类型图标、响应式标题、单行预览、使用次数、通用徽标、类型标签
  - 头部：当前项目胶囊下拉 + 显式搜索提示（placeholder 提示 `#tag`）+ 固定按钮 + 设置按钮
  - 鼠标：整行都是可点击命中区域，单击任意位置即执行，适合快捷粘贴板的高频路径
  - 底部键位提示：↑↓ / ⏎ / ⌘C / ⌘1-9 / ⌘P / Esc + 条数
- 词条新增 `tags: [String]` 字段
  - 新增迁移 `v4_entry_tags`，列类型 `TEXT NOT NULL DEFAULT '[]'`，内容为 JSON 字符串数组
  - `Entry` 模型改为手写 `FetchableRecord/PersistableRecord`，支持 `[String] <-> JSON` 编解码
  - 词条编辑表单新增「标签」输入框（逗号分隔，支持中文逗号/顿号/空格切分）
  - 内容库筛选芯片新增「高频标签」 Top 8
  - 预览底部新增标签列表
  - 快捷面板支持 `#tag` 搜索语法（FTS 结果上做二次过滤）
- 快捷面板键盘交互补齐
  - `⌘1..⌘9` 直达：按键即执行第 N 条可见词条
  - `⌘C`：仅复制当前选中词条并关闭面板，不走自动粘贴
  - 搜索框内有文本选区时 `⌘C` 保留系统默认（不冲突）
- 清理：删除 `Features/{Entries,Permissions,Projects,Settings}/` 空目录、`docs/ui-qa/latest/` 旧截图、`Constants.PanelLayout`/`Constants.Interface` 未再使用的布局常量

## 2026-04-24 效率极简风格收束（本轮）

本轮不改数据模型和执行主链路，重点是把主窗口、快捷面板和设置页进一步收成统一的“效率极简”风格。

- 统一视觉基线
  - 在 `Constants.swift` 新增统一密度/间距 token，收口主用行高、控件高度、section 圆角、badge 圆角和标题栏/底栏高度
  - 继续沿用现有深浅色动态配色，但减少正常态的大块强调色，主要靠文字层级、微弱底色和 hairline divider 建立结构
- 快捷面板
  - 头部继续只保留项目切换、搜索、设置三个入口
  - 正常态不再显示状态条；仅在权限异常或搜索/保存失败时显示单行异常提示
  - 结果行继续保留序号、来源标识、类型标识和数字直达，但右侧元信息更轻，选中态改为更弱的单层高亮
  - 底部提示栏保留开关，但默认只显示最关键的键位：`↑↓ / Enter / Esc / ⌘C / ⌘1-9`
- 内容库
  - 继续采用“三栏结构”，但左侧项目栏底部不再堆放重命名/删除按钮，只保留“设为当前项目”这个强相关动作；其余维护动作继续放到右键菜单
  - 中栏继续按“搜索 -> 筛选 -> 排序 -> 浏览”展开，工具带压缩成一行
  - 词条列表与预览区继续保持现有功能，但列表行、元信息和底部标签展示更紧凑、更低噪
- 设置页
  - 继续按“左侧高频设置 / 右侧运行概况与维护 / 底部数据与日志”组织，不改功能分组
  - 新增安静的顶部健康摘要：正常态只给一行概览；权限未授权、没有备份或近 7 天执行失败才升成明确提示
  - 运行概况中的执行统计块改为更轻的密度；数据位置与最近执行记录继续保留完整信息，但压缩行高和留白
