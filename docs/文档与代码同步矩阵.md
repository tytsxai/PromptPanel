# PromptPanel 文档与代码同步矩阵

更新时间：`2026-05-07`

本文是维护文档体系的操作规则。任何代码、脚本、配置、发布或 UI 基准变更，都要按下面矩阵同步文档，避免接手人看到的是旧系统。

## 1. 总原则

- 文档必须描述当前代码和脚本的真实行为，不写“未来计划”冒充现状。
- `docs/` 当前随仓库版本化，交接时要把文档同步视为代码交付的一部分。
- 如果文档和源码冲突，以源码 / 脚本 / 真实运行证据为准，然后立即修文档。
- 每次改完代码，至少搜索一次相关关键词，确认 README、docs、脚本说明没有旧说法。
- 每次改完代码、脚本、CI、配置、发布流程或 UI 基准，都必须跑 `scripts/check-docs.sh`；它是仓库当前的文档同步门禁。
- 发布、备份、恢复、权限和数据迁移属于高风险文档面，不能只改代码不改 runbook。

## 3. 变更后检查命令

基础检查：

```bash
git diff --stat
./scripts/check-docs.sh
swift build
```

本机有完整 Xcode 时：

```bash
swift test
./scripts/release-readiness.sh --output-dir /tmp/promptpanel-ready-check
```

本机只有 Command Line Tools 时，只能做降级预检：

```bash
swift build --build-tests
./scripts/release-readiness.sh --allow-build-only-tests --output-dir /tmp/promptpanel-ready-check
```

脚本和文档链接人工检查：

```bash
zsh -n scripts/build-app.sh
zsh -n scripts/check-docs.sh
zsh -n scripts/release-readiness.sh
zsh -n scripts/restore-backup.sh
zsh -n scripts/notarize-app.sh
zsh -n scripts/launch-computer-use.sh
```

说明：

- `scripts/check-docs.sh` 当前检查必备文档、关键链接、陈旧说法、`PROMPTPANEL_*` 环境变量覆盖、顶层维护脚本是否被文档提到，以及 CI 是否跑文档门禁。
- `promptpanel.sqlite` 是旧说法，当前数据库文件是 `promptpanel.db`。
- migration 链路当前包含 `v1` 到 `v5`，只写到 `v4` 就是过期。
- 默认热键当前是 `⌥2`，旧默认热键会被迁移。
- 缺少 `xctest` 时不能宣称完整测试执行通过。

## 4. 文档更新验收标准

完成一次文档同步时，至少满足：

- 新增 / 修改的运行行为在对应文档里能被找到。
- 文档里的命令在当前仓库路径下可执行或明确说明需要的外部条件。
- 发布和恢复文档没有跳过备份、签名、公证、恢复演练这些关键边界。
- 配置文档没有把 QA 环境变量当成长期生产配置。
- 文档没有把容器或普通服务器写成产品运行态。
- 如果本机验证只能降级，文档必须写清“build-only 不等于测试已执行”。

## 5. 什么时候必须更新回归记录

下面几类改动必须更新 [回归清单](./回归清单.md) 或 [兼容性回归记录](./兼容性回归记录.md)：

- 热键、面板激活、焦点恢复、固定面板、面板尺寸 / 位置。
- 自动粘贴、剪贴板兜底、权限引导、执行日志字段。
- 数据迁移、备份、恢复、日志清理。
- Sparkle、签名、公证、打包结构。
- `frontend-draft/` 和 SwiftUI/AppKit 可见 UI 的同步调整。
