## 已确认并执行删除

### 1. `.build/`

- 性质：Swift Package Manager 本地构建缓存、依赖工作副本和中间产物。
- 风险判断：不纳入 Git；执行 `swift build` / `swift test` 后可自动重建。
- 本次处理：已删除。

- 性质：本地打包输出目录，包含历史 `.app`、`.zip`、UI QA 构建产物、release-readiness 输出。
- 风险判断：脚本仅将其作为输出目录使用，不是源码或运行依赖；后续重新打包时会自动生成。
- 本次处理：已删除。

## 已验证

- 删除后执行 `swift test`，构建与测试恢复正常，证明上述目录属于可再生内容。
- 验证完成后再次清掉新生成的 `.build/`，确保空间真正释放。

- 原因：当前文档中将其定义为 UI 唯一基准，且目录内容已纳入版本库，不属于“低风险可直接清理”的范围。

- 原因：体积较小，且属于本地分析数据；当前没有足够证据证明它已彻底失效，暂不为节省极少空间而删除。
