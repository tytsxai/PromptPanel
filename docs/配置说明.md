# PromptPanel 配置说明

更新时间：`2026-05-07`

## 1. 配置分层

PromptPanel 当前配置分五层：

1. **编译期固定常量**
2. **`Info.plist` / entitlements**
3. **打包脚本注入参数**
4. **运行时环境变量**
5. **用户持久化设置**

接手时先判断一个配置到底属于哪一层，不要混着改。

## 2. 编译期固定常量

主要定义在 `Sources/PromptPanel/Core/Utils/Constants.swift`。

| 配置 | 当前值 / 规则 | 说明 |
| --- | --- | --- |
| 应用名 | `PromptPanel` | 代码层名称 |
| Bundle Identifier | `com.promptpanel.app` | 包标识 |
| 默认项目 | `通用项目` | 首次初始化时写入数据库 |
| 自动启动备份保留数 | `7` | 仅针对 `launch` 备份 |
| 自动启动备份最小间隔 | `12 小时` | 避免每次启动都生成新备份 |
| 执行日志保留天数 | `30 天` | 启动维护和手动清理共用 |
| 面板热键默认值 | `⌥2` | `HotkeyService` 会迁移历史默认值 |
| 面板默认内容尺寸 | `780 × 440` | 匹配 `frontend-draft/index.html` 的快捷面板基准 |
| 面板内容尺寸范围 | `560 × 300` 到 `1120 × 760` | 设置页写入时会归一化到这个范围 |
| 面板打开耗时目标 | `300ms` | 超过会写慢样本日志 |
| 搜索耗时目标 | `80ms` | 超过会告警 |
| 执行耗时目标 | `250ms` | 超过会告警 |
| 目标应用恢复超时 | `700ms` | 超时后进入焦点恢复失败分支 |

这些值改动后，需要同步更新：

- [架构说明](./架构说明.md)
- [关键模块与核心逻辑](./关键模块与核心逻辑.md)
- 必要时更新 [回归清单](./回归清单.md)

## 3. `Info.plist` 与 entitlements

资源文件位于：

- `Sources/PromptPanel/Resources/Info.plist`
- `Sources/PromptPanel/Resources/PromptPanel.entitlements`

### 3.1 当前关键项

| 键 | 当前值 | 说明 |
| --- | --- | --- |
| `CFBundleDisplayName` | `项目快贴` | 对用户显示的名称 |
| `CFBundleIdentifier` | `com.promptpanel.app` | 包标识 |
| `CFBundleShortVersionString` | `1.0.0` | 短版本 |
| `CFBundleVersion` | `1` | build 号 |
| `LSMinimumSystemVersion` | `14.0` | 最低系统版本 |
| `LSUIElement` | `true` | 菜单栏常驻、无 Dock 图标 |

### 3.2 entitlements

当前只有：

- `com.apple.security.app-sandbox = false`

也就是说：

- 当前不是 App Sandbox 应用
- 数据目录和系统集成逻辑按非沙盒桌面应用设计

## 4. 打包脚本参数

### 4.1 `scripts/build-app.sh`

| 参数 | 作用 |
| --- | --- |
| `--debug` | 生成 debug 包 |
| `--output-dir <path>` | 指定输出目录 |
| `--sign-identity <id>` | 指定签名证书；传 `none` 时做 ad-hoc 签名 |
| `--no-archive` | 不生成 zip |
| `--short-version <ver>` | 覆盖 `CFBundleShortVersionString` |
| `--build-version <ver>` | 覆盖 `CFBundleVersion` |
| `--bundle-identifier <id>` | 覆盖打包产物的 `CFBundleIdentifier` |
| `--display-name <name>` | 覆盖打包产物的 `CFBundleDisplayName` / `CFBundleName` |
| `--sparkle-feed-url <url>` | 注入 `SUFeedURL` |
| `--sparkle-public-ed-key <key>` | 注入 `SUPublicEDKey` |

补充说明：

- `--bundle-identifier` / `--display-name` 只影响打包产物，适合隔离 QA、`Computer Use` 或并行验收。
- `--sparkle-feed-url` 和 `--sparkle-public-ed-key` 必须成对提供；只提供其中一个时脚本会直接失败，避免生成“看似有更新配置、实际不可用”的发布包。
- 正式发布仍应保持默认包标识，不要把临时 QA 标识带到正式产物里。

### 4.2 `scripts/release-readiness.sh`

| 参数 | 作用 |
| --- | --- |
| `--output-dir <path>` | 发布前检查产物目录 |
| `--sign-identity <id>` | 传递给打包脚本 |
| `--allow-build-only-tests` | 当前机器缺少 `xctest` 时显式降级为测试包构建校验 |
| `--public-distribution` | 启用对外发包硬门槛检查 |
| `--skip-smoke-launch` | 跳过隔离启动冒烟 |
| `--short-version <ver>` | 覆盖版本号 |
| `--build-version <ver>` | 覆盖 build 号 |
| `--sparkle-feed-url <url>` | 注入 Sparkle feed |
| `--sparkle-public-ed-key <key>` | 注入 Sparkle 公钥 |

### 4.3 `scripts/notarize-app.sh`

| 参数 | 作用 |
| --- | --- |
| `--app-path <path>` | 已签名 `.app` 路径，必填 |
| `--archive-path <path>` | 提交给 Apple 的 zip/dmg/pkg；不传会临时生成 zip |
| `--keychain-profile <name>` | `notarytool` keychain profile，必填 |
| `--skip-staple` | 跳过 staple |
| `--skip-assess` | 跳过 Gatekeeper assess |

## 5. 运行时环境变量

### 5.1 数据与日志目录

| 环境变量 | 作用 | 常用场景 |
| --- | --- | --- |
| `PROMPTPANEL_APP_SUPPORT_DIR` | 覆盖应用数据根目录 | 隔离 QA、回归测试、临时验收 |
| `PROMPTPANEL_LOGS_DIR` | 覆盖日志目录 | 隔离 QA、收集排障日志 |

如果不传：

- 数据目录默认是 `~/Library/Application Support/PromptPanel`
- 日志目录默认是 `~/Library/Logs/PromptPanel`

### 5.2 QA / 调试

| 环境变量 | 作用 | 默认 |
| --- | --- | --- |
| `PROMPTPANEL_QA_OPEN_PANEL_ON_LAUNCH` | 启动后自动拉起面板 | 关闭 |
| `PROMPTPANEL_QA_OPEN_PANEL_DELAY_MS` | 自动拉起延迟毫秒数 | `700` |
| `PROMPTPANEL_QA_OPEN_MAIN_WINDOW_ON_LAUNCH` | 启动后自动打开主窗口 | 关闭 |
| `PROMPTPANEL_QA_OPEN_MAIN_WINDOW_DELAY_MS` | 主窗口自动打开延迟毫秒数 | `500` |
| `PROMPTPANEL_QA_MAIN_WINDOW_TAB` | 自动打开主窗口时进入 `library` 或 `settings` | 空，默认内容库 |
| `PROMPTPANEL_ALLOW_EXISTING_INSTANCE` | 跳过同 bundle id 单实例保护 | 关闭，仅限隔离 QA / release smoke |
| `PROMPTPANEL_ERASE_ON_SCHEMA_CHANGE` | schema 变化时允许擦库重建 | 关闭，且只应在明确破坏性调试时用 |

`PROMPTPANEL_ERASE_ON_SCHEMA_CHANGE` 是高风险开关：

- 仅适合开发调试
- 默认不该在正常升级或验收环境启用

### 5.3 Sparkle 更新配置

| 环境变量 | 作用 | 优先级 |
| --- | --- | --- |
| `PROMPTPANEL_SPARKLE_FEED_URL` | 指定 appcast feed URL | 高于 `Info.plist` |
| `PROMPTPANEL_SPARKLE_PUBLIC_ED_KEY` | 指定 Sparkle EdDSA 公钥 | 高于 `Info.plist` |

当前解析优先级是：

1. 环境变量
2. `Info.plist`

只有 feed URL 和 public key 同时存在时，Sparkle 才真正启用。

打包和发布门禁也执行同一条边界：`scripts/build-app.sh` 与 `scripts/release-readiness.sh` 都会拒绝只传 `SUFeedURL` 或只传 `SUPublicEDKey` 的半配置参数；对外发包前必须保证两者来自同一套 appcast 配置。

## 6. 用户持久化设置

PromptPanel 当前的用户设置并不集中在单一配置文件，而是分布在两类位置：

### 6.1 SQLite `app_settings`

当前明确持久化的项目级设置包括：

- `current_project_id`
- `panel_pinned`
- `panel_content_width`
- `panel_content_height`
- `panel_window_origin_x`
- `panel_window_origin_y`
- `panel_show_footer`
- `panel_compact_rows`
- `app_theme`
- `entry_sort_mode`

### 6.2 系统 / 第三方组件持久化

| 设置 | 存放位置 | 说明 |
| --- | --- | --- |
| 全局快捷键 | `KeyboardShortcuts` 自身持久化 | 不是 `app_settings` 表的一部分 |
| 登录启动 | `SMAppService` 系统注册状态 | 受系统设置影响 |

因此排查“设置为什么没生效”时，先确认它属于数据库态、系统态还是组件态。

## 7. 本地数据目录结构

默认目录结构如下：

```text
~/Library/Application Support/PromptPanel/
├── promptpanel.db
├── promptpanel.db-wal
├── promptpanel.db-shm
├── Backups/
│   ├── promptpanel-<timestamp>-launch-<id>.sqlite
│   └── promptpanel-<timestamp>-manual-<id>.sqlite
└── Recovery/
    ├── recovered-<timestamp>/
    └── manual-restore-<timestamp>/
```

日志目录：

```text
~/Library/Logs/PromptPanel/
```

## 8. 配置变更的推荐顺序

### 改版本号 / 构建号

优先用脚本参数覆盖，不建议直接手改 `Info.plist` 再忘记恢复。

### 改 Sparkle 更新地址

优先在打包阶段注入，或者在受控 QA 环境用环境变量覆盖。

### 改数据目录

优先用 `PROMPTPANEL_APP_SUPPORT_DIR` 做隔离，不要手工改代码常量。

### 改热键默认值

除了代码改默认值，还要考虑：

- 旧默认值迁移
- 文档同步
- 回归清单更新

### 改 SwiftPM 依赖版本

优先先确认 GitHub Actions 当前 Xcode / Swift 版本能解析并构建。`Package.swift` 里 `GRDB.swift` 当前限制在 `7.0.0..<7.9.0`，是为了和 `macos-14` / Xcode 16.2 的 CI 工具链保持一致；放宽到更新版本前，必须先在远端 CI 跑通。

## 9. 配置相关常见误区

- `Info.plist` 里的 `CFBundleExecutable` 在打包时会被 `build-app.sh` 重写，不要只看源文件就下结论。
- 本机新 Swift 能解析更新依赖，不代表 GitHub Actions 当前 Xcode 也能解析；依赖范围以 CI 可复现为准。
- Sparkle 依赖不是接上就等于“自动更新可用”；没有 feed URL 和公钥时，更新器会保持禁用。
- 打包阶段的 Sparkle feed URL 和 EdDSA 公钥必须成对注入；半套配置不是降级能力，而是发布配置错误。
- `swift test` 能否真正执行，除了项目配置，还取决于本机是否有 `xctest` runner。
- 不能把环境变量覆盖当作永久配置；它主要用于 QA、隔离测试和临时发布验证。
- `PROMPTPANEL_ALLOW_EXISTING_INSTANCE=1` 只用于隔离启动和自动化验收，正式运行时不要打开，否则可能绕过单实例保护。
